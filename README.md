# FinalProject
Wikipedia Web Traffic (Kaggle) Ежедневное количество просмотров статей Википедии за 2018-2019 годы.
Проект посвящён комплексному анализу временных рядов посещений страниц Wikipedia с применением методов машинного обучения для прогнозирования трафика и обнаружения аномалий.
Датасет
Источник: Wikipedia Web Traffic (Kaggle)
Формат: CSV файл с колонкой Page и ежедневными значениями трафика
Структура: строки — страницы Wikipedia, столбцы — даты
Период: 2015-2016 годы
Особенности: наличие пропущенных значений, различная популярность страниц
Выполненные этапы
1. Исследовательский анализ данных (EDA)
Проведён анализ структуры данных и статистических характеристик:
Распределение трафика по всем страницам
Анализ недельной сезонности (трафик по дням недели)
Визуализация общей динамики трафика во времени
Сравнение распределений трафика для топ-10 самых популярных страниц
2. Предобработка данных
Выполнена подготовка данных для моделей машинного обучения:
Интерполяция пропущенных значений (линейная интерполяция)
Создание временных признаков:
День недели, день месяца, месяц, день года
Флаг выходного дня
Создание лаговых признаков (lag 1, 3, 7, 14, 30 дней)
Расчёт скользящих статистик (MA_7, MA_30, Std_7)
Хронологическое разделение на обучающую (85%) и тестовую (15%) выборки
Масштабирование признаков для линейных моделей
3. Прогнозирование временного ряда
Применено 5 подходов для прогнозирования трафика:
Классические ML модели:
Linear Regression — линейная регрессия с масштабированными признаками
Random Forest — ансамбль решающих деревьев (100 деревьев)
SVR — метод опорных векторов для регрессии (RBF ядро)
Статистические модели временных рядов:
ARIMA (5,1,1) — авторегрессионная интегрированная скользящая средняя
Holt-Winters — экспоненциальное сглаживание с трендом и сезонностью (период 7 дней)
Оценка качества: RMSE, MAE, R² на тестовой выборке
4. Поиск аномалий
Применено 3 метода обнаружения аномалий:
Isolation Forest — изолирующий лес (contamination=5%)
Z-score — отклонения более 3 стандартных отклонений от остатков модели
IQR метод — межквартильный размах (границы Q1-3×IQR, Q3+3×IQR)
5. Визуализация результатов
Созданы графики для интерпретации результатов:
Сравнение фактических и прогнозируемых значений для всех моделей
Распределение ошибок прогнозирования (residuals)
Сравнение метрик качества моделей (RMSE, MAE, R²)
Важность признаков (feature importance) для Random Forest
Визуализация обнаруженных аномалий на фоне прогноза
Технические детали
Используемые библиотеки:
pandas, numpy — обработка данных
matplotlib, seaborn — визуализация
scikit-learn — ML модели, метрики, масштабирование
statsmodels — ARIMA, Holt-Winters
Архитектура решения:
Работа с одной репрезентативной страницей для демонстрации TS-моделей
Хронологическое разделение для предотвращения утечки данных
Модульная структура кода с разделением на логические блоки
Ключевые результаты
Лучшая модель: Random Forest показал наименьшие значения RMSE и MAE среди всех подходов
Важные признаки: лаговые признаки (Lag_1, Lag_7) и скользящие средние оказались наиболее информативными
Аномалии: Isolation Forest обнаружил наибольшее количество аномальных наблюдений (5% данных)
Сезонность: данные демонстрируют выраженную недельную сезонность
Практическая значимость
Проект демонстрирует полный цикл анализа временных рядов: от загрузки сырых данных до интерпретации результатов. Подходы могут быть применены для:
Прогнозирования нагрузки на веб-серверы
Обнаружения аномальной активности (DDoS-атаки, боты)
Оптимизации ресурсов хостинга
Анализа популярности контента
